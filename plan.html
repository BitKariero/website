Plan for Next Term

1 Outline for Next Term


  1. Milestone 1 (9th January – 23rd January 2017)
    1. API for creating Accounts
      • Should be able to requesting/providing records of multiple types.
      • Should be able to list all records assosiated with a particular profile.
    2. Documentation for Milestone 1.
  2. Milestone 2 (23th January – 6rd February 2017)
    1. API for record verification system and third-party identity vouching.
    2. Create Web Application UI for Milestone 1.
    3. Documentation for Milestone 2.
  3. Milestone 3 (6th February- 20th February 2017)
    1. API for floating records and general account management.
    2. UI for Milestone 1 and Milestones 2.
    3. Documentation for Milestones 3.
  4. Milestone 4 (20th February – 6th March 2017)
    1. API for creating publishable CVs (save to file)
    2. UI for Milestone 4.

2 Test Strategy


We plan to carry out testing during the development phase. We will use unit testing for the smart contracts and UI, potentially using a framework such as Embark. We will also carry out additional testing after each milestone, to try and resolve bugs before they become larger issues.
